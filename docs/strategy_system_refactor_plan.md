# 短线操盘实战改造方案

## 一、书中交易系统总纲

《短线操盘实战技法》的核心不是“堆很多形态”，而是建立一套有先后顺序的短线交易系统。按照书中内容，可提炼为以下主线：

1. **先看大盘环境**
   - 大盘强弱先决定是否出手。
   - 指数站稳关键均线、市场赚钱效应改善时，才提高出手频率。
   - 若大盘弱、情绪差，则宁可空仓或只做轻仓试错。

2. **再看热点板块与市场主线**
   - 板块是个股上涨的“势”。
   - 优先做热点题材、强势板块、资金集中方向。
   - 弱板块、冷门板块里的个股，即便形态不错，也降低优先级。

3. **在热点中选强势股**
   - 优先强于板块、强于大盘的个股。
   - 强势股、龙头股、率先突破股，优先于跟风股。
   - 选股核心是“强者恒强”与“由弱转强”的切换点。

4. **技术确认后再找买点**
   书中高频可执行买点主要包括：
   - 底背离 + 双突破
   - 低位123结构突破
   - 均线/趋势线突破
   - 支撑位回踩确认
   - 跳空缺口/涨停后的结构性突破

5. **仓位管理：轻仓试错，顺势加仓**
   - 首次买入不宜过重。
   - 只有走出来、确认正确后，才允许加仓。
   - 没有环境、没有板块、没有确认买点，不应重仓。

6. **止损必须刚性执行**
   - 结构低点止损
   - 支撑位止损
   - 缺口回补止损
   - 均线失守止损
   止损是交易前置条件，不是买入后的补充说明。

7. **止盈强调分批兑现与趋势跟随**
   - 先兑现部分利润，保住安全垫。
   - 剩余仓位让趋势去跑。
   - 出现放量滞涨、跌破关键支撑、情绪退潮时及时收缩。

8. **空仓也是交易系统的一部分**
   - 书中不是鼓励时时出手，而是强调“符合条件才做”。
   - 大盘弱、板块弱、个股弱时，空仓等待就是正确动作。

## 二、项目现状与书中体系的核心差距

当前项目已经具备较强的局部能力：

- 有 YAML 策略引擎
- 有底背离、123、缺口、涨停、趋势线、均线等检测器
- 有热点题材增强与龙头评分
- 有 AI 二次分析

但整体仍偏向：

> 多个选股器并列运行 + AI 辅助解释

而不是：

> 大盘环境 → 板块主线 → 强势股池 → 买点识别 → 风控执行

核心问题：

1. **重形态、轻流程**：有很多好形态，但没有明确交易阶段。
2. **环境层不够强**：大盘/板块没有成为所有策略的前置总开关。
3. **选股、买点、风控混杂**：多个策略各自顺手给一点买点和风控，缺统一口径。
4. **AI 二筛更像解释器，不像系统裁判。**
5. **异体系策略混入较多**：缠论、波浪、箱体、情绪周期等，会冲淡书中主线。

## 三、按照书中内容建议的系统分层

### Layer 1：市场环境层
职责：先回答“今天能不能做、该怎么做”。

输出建议：
- aggressive（进攻）
- balanced（均衡）
- defensive（防守）
- stand_aside（空仓/观望）

### Layer 2：板块/题材层
职责：识别市场主线和热点方向，将外部热点信息映射到 A 股本地板块体系，完成"题材 → 板块 → 个股"的关联链路。

#### 2.1 数据来源：双通道输入

| 通道 | 来源 | 说明 |
|------|------|------|
| 外部热点 | OpenClaw 盘前简报 | 通过 `POST /api/v1/screening/openclaw-theme-run` 传入，包含题材名、heat_score、confidence、keywords、evidence |
| 本地板块 | BoardSyncService → SQLite | 从 efinance 同步全市场个股所属板块（行业+概念），存入 `board_memberships` 表 |

外部热点是"语义化的市场情绪信号"（如"锂电池/锂矿"、"AI Agent/大模型"），本地板块是"A 股交易所/数据商定义的板块名"（如"锂矿概念"、"AIGC概念"）。Layer 2 的核心任务就是**把两者对齐**。

#### 2.2 题材归一化算法（ThemeNormalizationService）

从 OpenClaw 传入的题材名往往是自然语言描述（"创新药出海"、"被动元器件涨价"），需要归一化到本地板块词表。流程分三层：

**Step 1：拆分复合题材**
- 按 `/`、`／`、`、` 分割，如 `"锂电池/锂矿"` → `["锂电池", "锂矿"]`
- 去重、去空格

**Step 2：别名词表精确匹配**
- 维护一份人工校准的映射词表 `data/theme_aliases.json`
- 结构：`raw_alias → { normalized_label, matched_boards[] }`
- 例：`"创新药出海"` → `matched_boards: ["创新药"]`
- 例：`"锂价反弹"` → `matched_boards: ["锂矿概念", "锂电池概念"]`
- 命中别名 → 信心值 = 1.0（high_confidence）

**Step 3：候选板块模糊召回（BoardCandidateRecallService）**
- 别名未命中时，从本地全量板块名词表中做三层模糊匹配：

| 层级 | 条件 | 分数 |
|------|------|------|
| 精确命中 | `theme_part == board_name`（忽略大小写） | 1.0 |
| 子串包含 | `theme_part ⊂ board_name` 或 `board_name ⊂ theme_part` | 0.7 |
| 关键词重叠 | OpenClaw keywords 中任一关键词出现在 board_name 中 | 0.5 |

- 按分数降序取 top_k=5 候选板块
- 命中召回 → 信心值 = 0.6（weak_match）
- 若别名 + 召回同时有结果 → 信心值 = 0.9

**Step 4：最终输出**
每个原始题材归一化后产出：
```python
{
    "raw_theme": "锂电池/锂矿",         # 原始题材名
    "normalized_label": "锂电池/锂矿概念",  # 归一化标签
    "matched_boards": ["锂电池", "锂电池概念", "锂矿概念"],  # 映射到的本地板块
    "match_confidence": 1.0,             # 信心值
    "match_reasons": ["alias_hit"],      # 命中原因
    "status": "high_confidence"          # high_confidence / weak_match / unresolved
}
```

#### 2.3 个股-题材匹配算法（ThemeMatchingService）

归一化完成后，需要判断每只个股是否属于某个热点题材。使用**加权三因子打分**：

```
theme_match_score = board_match × 0.55 + name_match × 0.20 + keyword_match × 0.25
```

| 因子 | 权重 | 计算方式 |
|------|------|----------|
| board_match | 55% | 个股所属板块 vs 归一化后的板块名，取最高 SequenceMatcher 相似度 |
| name_match | 20% | 个股名称 vs 题材名，SequenceMatcher 相似度 |
| keyword_match | 25% | OpenClaw keywords 命中率 = 命中数 / 总关键词数（搜索空间含个股名 + 所有板块名） |

**硬门槛补偿规则：**
- 若 `board_match ≥ 0.95`（板块名几乎完全匹配），直接视为热点股
- 若 `board_match ≥ 0.75` 且 `keyword_match ≥ 0.50`，也视为热点股
- 目的：避免因个股名称与题材名差异太大，而错杀板块明确归属的个股

**热点股判定阈值：** `theme_match_score ≥ 0.80`

#### 2.4 龙头评分算法（LeaderScoreCalculator）

对通过热点门槛的个股，计算龙头评分（满分 100）：

| 维度 | 满分 | 评分逻辑 |
|------|------|----------|
| 题材匹配度 | 35 | `theme_match_score × 35` |
| 小市值加分 | 20 | 流通市值 < 500亿: 20分；500-1000亿: 10分；> 1000亿: 0 |
| 换手率 | 20 | > 5%: 20分；2-5%: 10分；< 2%: 0 |
| 突破强度 | 15 | 涨停+跳空缺口: 15分；仅一项: 10分；无: 0 |
| 趋势强度 | 10 | 站上MA100 + 近5日突破: 10分；仅站上MA100: 5分；无: 0 |

**龙头筛选阈值：** `leader_score ≥ 50`

#### 2.5 当前问题与改造方向

**已实现（能力清单）：**
- ✅ 外部热点题材接收（OpenClaw → ExternalTheme）
- ✅ 别名词表 + 模糊召回的归一化链路
- ✅ 个股板块匹配三因子打分
- ✅ 龙头评分五维度体系
- ✅ BoardSyncService 板块同步

**核心问题（需改造）：**

1. **板块强度缺失**：当前只看"个股是否属于热点板块"，没有计算"板块整体有多强"。应新增**板块强度指标**：
   - 板块涨幅排名（当日/近3日/近5日）
   - 板块内涨停家数 / 涨幅 > 5% 家数
   - 板块资金净流入排名
   - 输出：`板块强度分`（0-100），作为后续 Layer 3 选股的加权依据

2. **题材生命周期缺失**：当前只有单日快照，没有追踪题材持续性。应新增：
   - 题材连续出现天数
   - 题材 heat_score 趋势（上升/持平/衰退）
   - 输出：`题材阶段`（启动期 / 加速期 / 分歧期 / 退潮期）

3. **别名词表覆盖不足**：模糊召回效果受限于板块名本身的语义，新题材容易 unresolved。应建立：
   - 定期回顾 unresolved 日志，补充别名词表
   - 或考虑引入 embedding 做语义相似度兜底（Phase 2+）

4. **板块没有成为真正的"总开关"**：当前板块匹配更像"加分器"，未来应改为：
   - Layer 2 输出"今日值得关注的 N 个板块"及其强度分
   - Layer 3 只在这些板块内选股，板块外个股不进入强势股池
   - 实现"先板块后个股"的严格分层

输出建议：
- 热点题材列表（含归一化结果、信心值、match_reasons）
- 板块强度排名（当前缺失，需新增）
- 题材阶段标签（当前缺失，需新增）
- 龙头/核心股评分

### Layer 3：强势股池层
职责：把具备关注价值的股票先放进池子，而不是直接给买卖结论。

输出建议：
- 观察池
- 重点池
- 龙头池

### Layer 4：买点识别层
职责：在强势股池里进一步识别“次日重点盯盘买点”和“轻仓试错买点”。

输出建议：
- watch（观察）
- focus（重点盯盘）
- probe_entry（轻仓试错）
- add_on_strength（走强后加仓）
- reject（不参与）

### Layer 5：交易管理层
职责：统一仓位、止损、止盈、空仓规则。

输出建议：
- 初始仓位
- 加仓条件
- 止损依据
- 第一止盈位 / 第二止盈位
- 失效条件

## 四、现有策略分层结论

### A类：保留，作为书中体系核心策略

1. **bottom_divergence_double_breakout**
   - 定位：反转型核心买点
   - 原因：高度符合“底背离 + 双突破”主线

2. **ma100_low123_combined**
   - 定位：右侧确认型核心买点
   - 原因：符合低位结构突破 + 趋势确认 + 买点时效控制

3. **ma100_60min_combined**
   - 定位：多周期入场策略
   - 原因：符合“看长做短、大级别选股小级别入场”

4. **gap_limitup_breakout**
   - 定位：强势启动型买点
   - 原因：符合书中对突破性缺口、涨停突破的重视

5. **extreme_strength_combo**
   - 定位：热点强势股/龙头池构建器
   - 原因：符合热点板块 + 龙头 + 强势共振思路
   - 注意：后续应拆层，不宜继续作为过胖总策略

6. **bull_trend**
   - 定位：基础趋势强势股池
   - 原因：书中根基就是做趋势、做强势、避免逆势硬做

7. **shrink_pullback**
   - 定位：趋势延续中的稳健买点
   - 原因：符合“回踩支撑买，不追高”的重要原则

### B类：保留，但降级为辅助策略 / 观察池策略

1. **dragon_head**
   - 定位：板块龙头辅助评分
   - 原因：龙头思路正确，但更适合做加分器而非独立主策略

2. **volume_breakout**
   - 定位：突破有效性确认器
   - 原因：书中认可放量突破，但它更像通用确认模块，不应单独过重

3. **trendline_breakout**
   - 定位：结构突破辅助确认器
   - 原因：适合并入更强主策略，不适合长期独立扛主线

4. **bottom_volume**
   - 定位：底部异动观察信号
   - 原因：可用于观察池，但不应进入核心执行层

5. **one_yang_three_yin**
   - 定位：K线辅助加分项
   - 原因：可作为辅助形态，不是书中核心主线

### C类：建议删除

1. **box_oscillation**
   - 原因：与书中“做强势、做趋势、做突破”的主线冲突较大

2. **chan_theory**
   - 原因：缠论是另一套完整交易哲学，会污染当前系统范式

3. **wave_theory**
   - 原因：波浪理论主观性较强，不符合本书的简洁实战路线

4. **emotion_cycle**
   - 原因：抽象度过高，容易把系统带到另一套叙事，不利于收敛

5. **ma_golden_cross**
   - 原因：过于泛化，与 bull_trend / shrink_pullback / ma100_60min 等重复，且确认度偏弱

## 六、后续改造顺序建议

### 第一阶段：先做分层，不急着重写算法
1. 删除异体系策略
2. 给保留策略重新定角色
3. 建立市场环境层 / 板块层 / 股票池层 / 买点层 / 风控层
4. 输出结构改成：观察池 / 重点池 / 轻仓试错池

### 第二阶段：重写 AI 二筛为结构化裁判
AI 应固定回答：
- 当前环境是否允许交易
- 是否属于热点核心
- 当前交易阶段
- 买点是否成熟
- 风险等级与止损依据
- 仓位与止盈建议

### 第三阶段：建立统一交易管理器
统一处理：
- 初始仓位
- 加仓条件
- 止损规则
- 止盈规则
- 空仓规则

## 七、改造目标一句话总结

本次改造的目标不是继续堆策略，而是把现有项目收敛成：

> 一套符合《短线操盘实战技法》主线的、分层清晰、能先环境后个股、先强势后买点、先规则后AI的短线交易系统。
